Data: Raw facts and figures without context. Examples include numbers, dates, and strings.
Information: Processed data that is meaningful and useful. It often comes in the form of reports or summaries.
Metadata: Data about data, providing details such as data type, source, and structure. It helps in understanding and managing data.
Data Directory: A directory that stores metadata and information about the structure of databases. It acts as a catalog for users and applications.
Database: A structured collection of related data stored electronically. Databases facilitate data management and retrieval.
Table: An organized collection of data in rows and columns within a database. Each table holds data about a specific type of entity.
Tuple(s): A single row in a table, representing a single record. Each tuple contains related data about an entity.
Field(s): A column in a table representing a specific attribute. Fields hold data values of a particular type, such as text or numbers.
Domain(s): The set of possible values for a given attribute or field. Domains ensure data integrity by restricting values to a predefined set.
Constraint(s): Rules applied to data to enforce integrity and accuracy. Constraints include primary keys, foreign keys, and unique constraints.
Key(s): Unique identifiers for records in a table. Primary keys uniquely identify each record, while foreign keys link related tables.
